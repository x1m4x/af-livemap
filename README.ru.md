# AF LiveMap

Живая карта для [Abiotic Factor](https://store.steampowered.com/app/427410/Abiotic_Factor/), которая **строит себя сама, пока ты играешь**. Мод пускает из камеры игрока лучи-лидар, из попаданий собирается облако точек, а локальный веб-сервер показывает карту в браузере — на ПК и на телефоне — в реальном времени.

Никаких готовых карт не нужно: ты просто ходишь и осматриваешься, а карта проявляется сама. По желанию сканирование привязывается к внутриигровому предмету **Rat Scanner**.

```
   Игра (UE4SS Lua-мод)              Локальный сервер (Python)          Браузер (ПК / телефон)
  позиции + лидар-лучи      ──▶     воксельная карта, точки,     ──▶    2D-карта и 3D-облако точек,
  пишутся в JSON-файлы              маршруты, синхронизация            метки, маршруты, заметки
```

---

## Что умеет

### 🗺️ Сканер карты — облако точек
- Мод каждый тик пускает из камеры десятки лучей по сфере (лидар). Точки попадания в стены копятся в воксельную сетку 50 см.
- Карта строится **от осматривания** — ходить вдоль каждой стенки не нужно, достаточно повертеть камерой.
- **Только накопление (по умолчанию):** точки не удаляются, стены не пропадают. Авто-очистку «призраков» (движущиеся машины/NPC) можно включить флагом сервера `--carve`.
- Скан **копится между сессиями** и общий для всех устройств.

### 🧭 Два вида
- **2D** — вид сверху. Цвет точки кодирует перепад высоты относительно игрока (свой уровень — ярко-голубой, выше — зелёный, ниже — синий), так читаются этажи и лестницы. Работает на любом удалении и высоте.
- **3D** — облако точек на WebGL: свободная орбитальная камера, раскраска по расстоянию до игрока, тысячи точек без тормозов. Мышь на ПК, тач на телефоне.

### 📍 Метки, лифты, тележки, порталы
- **Метки (вейпоинты)** — ставятся на позиции игрока, с названием. Переименование, удаление, «показать на карте», построение маршрута.
- **Лифты** — записываются одной кнопкой: встал в кабину, нажал «Лифт», прокатился по этажам, нажал ещё раз. Карта сама определяет остановки по высоте. Зона лифта исключается из скана (движущаяся кабина больше не мусорит), а маршруты умеют ездить между этажами на лифте.
- **Тележки / рельсовый транспорт** — записывается весь путь; в маршрутах это ребро «начало ↔ конец», причём рисуется реальная траектория рельсов.
- **Порталы — автоопределение.** Телепортировался (резкий скачок или смена мира) — портал записывается сам, вход и выход. Повторные проходы подтверждают его. Смерть/возрождение порталом не считается. Есть **зоны «не портал»** — для предмета-телепорта на базу, чтобы он не плодил ложные порталы.

### 🚶 Маршруты с учётом дверей
- Автоматический поиск пути (A*) по отсканированному «полу».
- **Двери учитываются сами:** сервер копит клетки, где игрок физически проходил, и считает их проходимыми поверх скана. Закрытая дверь выглядит в скане стеной — но если ты хоть раз через неё прошёл, маршрут через неё построится. Никаких кнопок.
- Лифты и тележки встроены в граф — маршрут сам выберет их, если так короче.

### 📝 Заметки и мультиплатформенность
- Общий блокнот с автосохранением, синхронизируется между ПК и телефоном.
- Сервер открыт в локальную сеть — открой карту на телефоне в том же Wi-Fi и смотри/редактируй оттуда. Интерфейс адаптирован под тач.
- Автопереключение на мир, где находится игрок; вручную можно смотреть другие миры (их скан и метки).

## Установка

Нужны: **Windows**, установленная **Abiotic Factor**, **Python 3.10+**.

### 1. UE4SS

Поставь UE4SS для Abiotic Factor (сборка с конфигами под игру — [Nexus Mods](https://www.nexusmods.com/abioticfactor/mods/35)). Распаковывается в папку игры:

```
<папка игры>\AbioticFactor\Binaries\Win64\
```

Путь к игре у всех свой, например `C:\Games\AbioticFactor\...` или `C:\Program Files (x86)\Steam\steamapps\common\AbioticFactor\...`. После установки при запуске игры рядом появляется папка `ue4ss\`.

### 2. Мод

Скопируй **содержимое** папки `mod/` из этого репозитория в `...\Win64\ue4ss\Mods\AFLiveMap\`, чтобы получилось:

```
ue4ss\Mods\AFLiveMap\Scripts\main.lua
ue4ss\Mods\AFLiveMap\config.txt
ue4ss\Mods\AFLiveMap\enabled.txt
```

`enabled.txt` включает мод без правки `mods.txt`. После запуска игры мод начнёт писать `livemap.json` и `lidar.json` в свою папку. В `UE4SS.log` появится строка `[AFLiveMap] Мод загружен...`.

### 3. Сервер

> Файлы сервера в папку игры класть **не надо**. Держи скачанную папку где угодно
> (хоть на рабочем столе) — ты просто запускаешь из неё маленькую Python-программу.
> Нужен установленный **Python 3.10+** ([python.org](https://www.python.org/downloads/),
> при установке отметь галочку **«Add Python to PATH»**).

**Проще всего — запусти `start-server.py`** (двойной клик или `python start-server.py`).
Он сам определит папку игры и запустит сервер. Если игру не найдёт — попросит один
раз вставить путь к `livemap.json` и запомнит его.

**Вручную (если удобнее командная строка):** из этой папки запусти —

```powershell
python server\server.py --data "C:\Games\AbioticFactor\AbioticFactor\Binaries\Win64\ue4ss\Mods\AFLiveMap\livemap.json"
```

**Путь `--data` подставь свой** — это `livemap.json` внутри папки мода из шага 2 (`...\ue4ss\Mods\AFLiveMap\livemap.json`). Если путь неверный, сервер честно напишет «ВНИМАНИЕ: файл не найден».

При старте сервер печатает два адреса:

```
AF LiveMap (этот ПК):  http://127.0.0.1:8765
AF LiveMap (телефон):  http://192.168.x.x:8765
```

Открой первый на ПК. Зайди в игру — статус сменится на «онлайн: 1 | мир: …», и карта начнёт строиться.

### 4. Телефон (по желанию)

Открой второй адрес (`http://192.168.x.x:8765`) на телефоне в том же Wi-Fi. При первом запуске Windows спросит про брандмауэр — разреши для частных сетей. Если диалога не было, добавь правило (терминал **от администратора**):

```powershell
netsh advfirewall firewall add rule name="AF LiveMap" dir=in action=allow protocol=TCP localport=8765
```

## Настройка (config.txt)

Файл `ue4ss\Mods\AFLiveMap\config.txt` — простой `ключ = значение`, редактируется блокнотом. После изменения перезапусти игру.

| Ключ | По умолчанию | Что делает |
|------|--------------|-----------|
| `require_rat_scanner` | `true` | `true` — карта строится **только когда держишь Rat Scanner** в руке. `false` — сканирование работает всегда, предмет не нужен. |
| `scanner_item_match` | `ratscanner` | Подстрока имени предмета в руке для опознания сканера (регистр не важен). Менять только если детект не срабатывает. |
| `log_held_item` | `false` | Диагностика: печатает имя предмета в руке в консоль UE4SS. Нужно, чтобы узнать точное `scanner_item_match`. |
| `language` | `auto` | Язык сообщений мода в логе: `auto` (по языку игры), `en`, `ru`. |
| `scan_enabled` | `true` | Полностью выключить лидар (позиции игроков всё равно пишутся). |
| `scan_rays` | `12` | Лучей за тик. Больше — быстрее карта, выше нагрузка. |
| `scan_range_cm` | `6000` | Дальность луча, см (6000 = 60 м). |

### Привязка к Rat Scanner

По умолчанию карта сканируется, **только пока в руке предмет Rat Scanner** — это делает картографию частью геймплея (нашёл сканер → можешь картографировать). Чтобы сканировать всегда, без предмета:

```
require_rat_scanner = false
```

Если с Rat Scanner в руке карта не строится — значит, у предмета другое внутреннее имя. Найди его так:
1. В `config.txt` поставь `log_held_item = true`, перезапусти игру.
2. Возьми Rat Scanner в руку, открой консоль UE4SS — там будет строка `Предмет в руке: ...`.
3. Впиши характерную часть этого имени в `scanner_item_match`, верни `log_held_item = false`, перезапусти.

## Карты секторов (по желанию)

Скан-карта (2D/3D) работает **без** каких-либо картинок. Кнопка «Карты» — это отдельный попап со скриншотами карт секторов из игрового журнала, чисто для справки; они в репозиторий не входят.

Чтобы добавить их: положи картинки (`.webp`/`.png`) в `maps/`, скопируй `maps/maps.example.json` → `maps/maps.json` и пропиши записи. Скриншоты карт секторов есть на [вики](https://abioticfactor.wiki.gg/) и в гайдах.

## Проверка без игры

Можно погонять сервер и веб без запущенной игры — симулятор пишет фейковые позиции и лидар:

```powershell
# терминал 1 — симулятор
python tools\fake_player.py --out livemap.json
# терминал 2 — сервер на этот файл
python server\server.py --data livemap.json
```

На карте будут ходить два фейковых игрока и накапливаться скан «комнаты».

## Как это устроено

- **Мод** (`mod/Scripts/main.lua`) — UE4SS Lua. В игровом потоке только лёгкий сбор данных (позиции + несколько трейсов), запись файлов и кодирование JSON вынесены в фоновый поток, чтобы не ронять FPS. Пишет `livemap.json` (позиции) и `lidar.json` (точки + откуда пущены лучи).
- **Сервер** (`server/server.py`) — Python stdlib, без зависимостей. Следит за файлами мода, копит воксельную карту и «прохоженные» клетки, чистит скан вдоль лучей, отдаёт веб и стримит обновления через SSE. Хранит точки/лифты/порталы/тележки/заметки в `data/`.
- **Веб** (`web/`) — чистый JS + Canvas (2D) и WebGL (3D), без сборки и библиотек.

Runtime-данные (`data/`), картинки секторов и файлы, которые пишет мод, в репозиторий не коммитятся (см. `.gitignore`) — они у каждого свои.
